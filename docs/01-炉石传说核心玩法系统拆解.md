## 一、回合系统

炉石传说采用 **固定回合交替制** ，双方轮流进行自己的回合，对手回合内不能主动执行任何操作（奥秘除外）。

### 1.1 回合的三个阶段

每个回合严格按以下顺序执行：

```
┌─────────────────────────────────────────────────────────────┐
│                    当前玩家的回合                             │
│                                                             │
│  [ 回合开始阶段 ] → [ 主要行动阶段 ] → [ 回合结束阶段 ]        │
│                                                             │
│  回合开始阶段不可打断       玩家自由操作        点击「结束回合」后进入
│  全部自动化处理             → 打出卡牌          → 触发结束阶段效果
│                             → 发起攻击          → 回合移交对方
│                             → 使用英雄技能
│                             → 表情互动
└─────────────────────────────────────────────────────────────┘
```

### 1.2 回合开始阶段（Start of Turn）

在主要行动阶段之前，按以下**精确顺序**自动执行：

| 步骤 | 动作 | 说明 |
|:---:|------|------|
| 1 | **获得法力水晶** | 当前回合数 ≤ 10 时，永久法力水晶 +1（上限 10）；若当前为第 1 回合，跳过此步骤（已有 1 颗） |
| 2 | **填满法力水晶** | 所有已消耗的法力水晶恢复为可用状态（包括临时获得的法力水晶，如幸运币） |
| 3 | **抽一张牌** | 从牌库顶抽 1 张牌加入手牌。**例外**：先手玩家第 1 回合不抽牌 |
| 4 | **触发「回合开始时」效果** | 场上所有具有"在你的回合开始时……"描述的随从/武器/奥秘效果按入场顺序依次触发 |

**关键细节**：
- 步骤 1-3 是**系统行为**，不产生事件触发
- 步骤 4 是**游戏事件**，可能触发其他卡牌效果（如奥秘、亡语等）
- 如果抽牌时牌库为空，触发**疲劳伤害**（见 1.6 节）

### 1.3 主要行动阶段（Main Phase）

玩家在此阶段可以自由执行以下操作（无固定顺序）：

| 操作 | 限制条件 |
|------|----------|
| **打出随从牌** | ① 有足够法力水晶 ② 己方战场随从数 < 7 |
| **打出法术牌** | ① 有足够法力水晶 ② 必须有合法的目标（如需指定目标） |
| **打出武器牌** | ① 有足够法力水晶 ② 替换当前武器（旧武器销毁） |
| **打出英雄牌** | ① 有足够法力水晶 ② 替换当前英雄及其英雄技能 |
| **使用英雄技能** | ① 有 ≥ 2 点法力水晶 ② 本回合尚未使用过英雄技能 |
| **随从攻击** | ① 随从攻击力 > 0 ② 随从已可以攻击（非召唤失调，或有冲锋/突袭）③ 本回合尚未攻击过（除非有风怒）④ 目标合法（无嘲讽时任意，有嘲讽时必须先攻击嘲讽随从） |
| **英雄攻击** | ① 英雄装备了武器且攻击力 > 0 ② 本回合尚未攻击过 |
| **点击「结束回合」** | 进入回合结束阶段 |

**操作次数限制**：
- 英雄技能：每回合最多 1 次（除非有特殊效果如"可以使用两次英雄技能"）
- 每张卡牌打出无次数限制（只要有足够法力水晶）
- 英雄攻击：每把武器每回合最多 1 次

### 1.4 回合结束阶段（End of Turn）

当玩家点击「结束回合」按钮后：

| 步骤 | 动作 | 说明 |
|:---:|------|------|
| 1 | **触发「回合结束时」效果** | 场上所有具有"在你的回合结束时……"描述的随从/效果按入场顺序依次触发 |
| 2 | **伤害/减益状态结算** | 某些"持续到你的回合结束"的临时效果在此刻移除 |
| 3 | **移除临时buff** | 如"本回合获得的攻击力"等在此时清零 |
| 4 | **回合移交** | 对方回合开始 |

**关键细节**：
- 步骤 1 中触发的效果**完全结算完毕后**，才进行步骤 2
- 如果回合结束阶段触发了某些效果导致随从死亡，亡语**在此刻**触发，不会等到下回合
- 回合结束阶段结束后，所有"本回合中……"类临时效果全部过期

### 1.5 回合计数

- 先手玩家回合编号为 1, 3, 5, 7……
- 后手玩家回合编号为 2, 4, 6, 8……
- 双方共享同一个递增的回合计数器（用于判断"第 X 回合"效果）
- 法力水晶数量 = min(回合编号, 10)

### 1.6 疲劳伤害（Fatigue）

当玩家需要抽牌但牌库已为空时：

```
疲劳计数器（每位玩家独立，初始为 0）：
  每次抽牌失败 → 疲劳计数器 +1 → 受到等于疲劳计数器当前值的伤害

示例：
  第 1 次疲劳 → 受 1 点伤害
  第 2 次疲劳 → 受 2 点伤害
  第 3 次疲劳 → 受 3 点伤害
  ……
```

## 二、法力水晶系统

### 2.1 法力水晶的生命周期

```
永久法力水晶（Permanent Mana Crystals）
  ├── 回合开始时，若当前回合数 ≤ 10，永久水晶 +1（最多 10 颗）
  ├── 每回合开始时，所有永久水晶重新填满（从空变为满）
  └── 消耗：打出卡牌 / 使用英雄技能时扣除
```

### 2.2 法力水晶的类型

| 类型 | 获取方式 | 特性 |
|------|----------|------|
| **永久法力水晶** | 每回合自动获得 1 颗（最多 10） | 每回合自动填满；不可被摧毁 |
| **临时法力水晶** | 卡牌效果（如幸运币、激活、滋养） | 仅当前回合有效；回合结束时消失；可超出 10 上限 |
| **过载法力水晶** | 某些卡牌打出时锁定下一回合的法力水晶 | 下回合开始时，指定数量的水晶被锁定为不可用 |

### 2.3 过载（Overload）机制

```
打出过载卡牌时：
  └── 记录过载值 X
      └── 下回合开始时：
          ├── 永久水晶正常填充
          ├── 但其中 X 颗被锁定（Locked），不可使用
          └── 锁定只影响下 1 回合，不可累积多回合
```

**过载的精确行为**：
- 过载锁定的是**下回合**的法力水晶，而非当前回合
- 如果连续多回合打出过载牌，过载量累加
- 过载只锁定永久水晶，不影响临时法力水晶
- 被锁定的水晶在回合界面显示为挂锁图标

### 2.4 幸运币（The Coin）

- 后手玩家开局获得 1 张幸运币，置于手牌中
- 幸运币是一张 0 费法术：「在本回合中，获得 1 个法力水晶」
- 幸运币获得的法力水晶是**临时**的，本回合结束时消失
- 幸运币占 1 张手牌位置（手牌上限 10 张）
- 幸运币属于法术牌，会触发法术相关效果（如法力浮龙 +1 攻击力）

## 三、卡牌类型体系

炉石共有 **4 种卡牌类型**：

### 3.1 随从（Minion）

随从是炉石最核心的卡牌类型。

**基础属性**：

| 属性 | 说明 |
|------|------|
| **法力值消耗** | 打出所需的法力水晶，显示在卡牌左上角 |
| **攻击力** | 每次对目标造成的伤害值，显示在卡牌左下角（黄色数字） |
| **生命值** | 能承受的伤害总量，显示在卡牌右下角（红色数字）；生命值降到 0 或以下时随从死亡 |
| **种族** | 随从的种族标签（野兽、龙、鱼人、机械、元素、海盗、恶魔、图腾、亡灵、纳迦），影响种族协同卡牌 |

**随从的两种状态**：

| 状态 | 说明 |
|------|------|
| **召唤失调（Summoning Sickness）** | 随从在被打出的当回合**不能攻击**（表现为卡牌周围有绿色 Zzz 特效），但可以触发其自身效果（战吼等） |
| **可攻击状态** | 从下一个自己的回合开始，随从解除召唤失调，可以攻击 |

**随从入场时的属性判定**：
- 随从打出后，生命值 = 基础生命值 + 光环buff + 战吼自身buff
- 攻击力 = 基础攻击力 + 光环buff + 战吼自身buff
- 随从打出后若生命值 ≤ 0，则**进场后立即死亡**

**随从死亡**：
- 生命值 ≤ 0 时，随从进入死亡状态
- 触发亡语（Deathrattle）
- 触发"随从死亡时"类效果
- 随从从战场移除，进入坟场（Graveyard）
- 不可复活被沉默后死亡的随从——此处详见关键词文档

### 3.2 法术（Spell）

**法术的特性**：
- 一次性效果，打出后立即生效并进入坟场
- 不占据战场位置
- 部分法术需要目标（以箭头指示），部分不需要
- 有法力值消耗，无攻击力/生命值

**法术的子类型**（通过关键词区分）：

| 子类型 | 说明 |
|--------|------|
| **普通法术** | 打出即生效，进入坟场 |
| **奥秘（Secret）** | 3 费及以下（通常），打出后挂在英雄头像上，对手看不到具体内容；当满足特定条件时**在对手回合**自动触发 |
| **任务（Quest）** | 1 费，必定在起始手牌中；打出后挂在英雄头像上，完成条件后获得奖励 |
| **支线任务（Sidequest）** | 类似任务但可通过发现等方式获得 |

**奥秘（Secret）的精确规则**：
- 每位玩家最多同时挂 5 个不同的奥秘（同名奥秘不可重复挂）
- 奥秘只能在**对手回合**触发（极少数例外）
- 多个奥秘满足条件时，按**挂载顺序**依次检查，最先满足的触发
- 奥秘触发后进入坟场
- 奥秘在己方回合永远不会触发（这是炉石铁律）

### 3.3 武器（Weapon）

**武器的双属性**：
| 属性 | 说明 |
|------|------|
| **攻击力** | 英雄攻击时造成的伤害 |
| **耐久度** | 可攻击的次数；每次攻击消耗 1 点，耐久度归零时武器销毁 |

**武器的规则**：
- 装备武器后，英雄获得攻击力（可在英雄头像旁看到攻击力数字）
- 英雄每回合最多用武器攻击 1 次
- 英雄攻击时，攻击动作与随从攻击相同：英雄受到目标攻击力的反击伤害
- 打出新武器时，旧武器自动销毁（不触发亡语——武器无亡语，除非特殊效果赋予）
- 武器在己方回合外无攻击力（英雄头像上的攻击力在对手回合显示为 0）
- 具有风怒的武器，英雄可攻击 2 次（如末日锤）

### 3.4 英雄牌（Hero Card）

**英雄牌的机制**：
- 打出后**替换**当前英雄
- 获得一定量的护甲值（5 点，通常）
- 触发战吼效果（如果有）
- **替换英雄技能**（新的英雄技能通常更强或不同）
- 英雄生命值不变（30 血 + 已有护甲不受影响，除非战吼改变了护甲量）

**英雄牌与其他卡牌类型的区别**：
- 英雄牌不是随从，不占战场格子
- 英雄牌打出后，英雄技能立即刷新为可用状态（即使本回合已用过旧英雄技能）

## 四、战场规则

### 4.1 战场布局

```
┌─────────────────────────────────────────────────────────────┐
│                      对手的信息区                             │
│  [英雄头像] [英雄技能] [武器] [奥秘×N] [牌库] [坟场]          │
│                                                             │
│  [随从1] [随从2] [随从3] [随从4] [随从5] [随从6] [随从7]      │ ← 对手战场
│                                                             │
│  ─────────────────── 分割线 ───────────────────              │
│                                                             │
│  [随从1] [随从2] [随从3] [随从4] [随从5] [随从6] [随从7]      │ ← 己方战场
│                                                             │
│  [英雄头像] [英雄技能] [武器] [奥秘×N] [牌库] [坟场]          │
│                      己方的信息区                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 战场容量

- 每方最多 **7 个随从**站位
- 随从按**打出顺序**从左到右排列
- 当战场已满（7 个随从）时，不能打出随从牌
- 随从死亡后，后续随从**不自动左移填补空位**
- 某些卡牌效果可通过"召唤"而非"打出"来增加随从，但如果战场已满则召唤失败
- **站位是固定的**，除非有卡牌效果移动随从

### 4.3 站位的重要性

炉石的站位并非纯视觉——它影响以下机制：

| 影响范围 | 说明 |
|----------|------|
| **相邻效果** | 某些卡牌效果作用于「相邻的随从」，站位决定哪些随从受影响 |
| **群体buff的顺序** | 如恐狼前锋：相邻随从 +1 攻击力，站位决定 buff 目标 |
| **战吼指定站位** | 部分卡牌允许打出时指定放在哪个位置（左右调整） |

### 4.4 坟场（Graveyard）

- 所有死亡的随从、使用过的法术、销毁的武器均进入坟场
- 坟场内容对双方可见（鼠标悬停查看）
- 坟场是复活效果的数据来源（如牧师复活术）
- 被沉默的随从死亡后仍在坟场中，复活后的随从**恢复其原始属性**（沉默效果被清除）

## 五、攻击与伤害结算

### 5.1 攻击的基本流程

```
玩家选择己方攻击者 → 选择敌方目标 → 执行攻击动画 → 结算伤害
```

**攻击的合法性检查**（按顺序）：

1. 攻击者是否已解除召唤失调？（随从）/ 是否装备了武器？（英雄）
2. 攻击者本回合是否已攻击过？（风怒单位可攻击 2 次）
3. 攻击者的攻击力是否 > 0？
4. 对方场上是否有**嘲讽（Taunt）随从**？
   - 有嘲讽 → 只能选择嘲讽随从作为攻击目标
   - 无嘲讽 → 可选择任何敌方随从或敌方英雄
5. 目标是否具有**潜行（Stealth）**？
   - 有潜行 → 不可被指定为攻击目标，不可被指向性法术/英雄技能指定
6. 攻击者是否被**冻结（Frozen）**？
   - 被冻结 → 本回合不可攻击（冻结在下回合开始时自动解除）
7. 目标是否具有**免疫（Immune）**？
   - 有免疫 → 不可被攻击

### 5.2 伤害结算（核心算法）

当一个攻击力为 A 的随从/英雄攻击一个攻击力为 B、生命值为 H 的目标时：

```
随从 攻击 随从：
  攻击者受到 B 点伤害（目标的反击伤害）
  目标受到 A 点伤害（攻击者的攻击伤害）
  双方伤害同时结算（同时发生，不分先后）

随从 攻击 英雄：
  英雄受到 A 点伤害
  英雄若未装备武器 → 攻击者不受伤害
  英雄若装备武器（攻击力为 W）→ 攻击者受到 W 点伤害

英雄 攻击 随从：
  英雄受到 B 点伤害（随从的反击伤害）
  随从受到 W 点伤害（英雄武器攻击力）
  英雄消耗 1 点武器耐久

英雄 攻击 英雄：
  攻击英雄受到 B' 点伤害（若对方有武器）
  目标英雄受到 W 点伤害
  攻击英雄消耗 1 点武器耐久
```

### 5.3 伤害结算中的特殊机制

**圣盾（Divine Shield）**：
- 具有圣盾的单位，第一次受到伤害时：伤害降为 0，圣盾消失
- 圣盾只挡**伤害**，不挡"消灭"效果（如术士的腐化迷雾）
- 随从攻击圣盾随从：攻击者不受伤（因为目标受到的伤害被吸收了），但圣盾消失

**剧毒（Poisonous）**：
- 具有剧毒的随从造成的**任何伤害**（包括 0 点伤害），目标随从立即被消灭
- 剧毒 + 圣盾交互：剧毒随从攻击圣盾随从 → 造成 0 伤害（圣盾吸收）→ 0 伤害仍触发剧毒 → 圣盾随从被消灭，圣盾随从不反击 → 剧毒随从不受伤
- 剧毒对英雄无效（只影响随从）

**吸血（Lifesteal）**：
- 具有吸血的随从/法术造成伤害时，同时为英雄恢复等量生命值
- 过量伤害也产生过量治疗（可以超出 30 血上限）

**风怒（Windfury）**：
- 具有风怒的单位每回合可攻击 2 次
- 第一次攻击后，单位状态变为"已攻击但可再攻击一次"
- 第二次攻击后，单位变为"已攻击"状态
- 风怒 + 武器：英雄可攻击 2 次，每次消耗 1 耐久

**冻结（Freeze）**：
- 被冻结的角色**跳过其下一次可攻击的机会**
- 冻结持续到下个己方回合开始时自动解除
- 如果角色本回合已经攻击过，冻结将阻止其**下回合**攻击
- 冻结不影响突袭的攻击允许性

**免疫（Immune）**：
- 具有免疫的角色不能成为攻击目标，也不能被指定为目标
- 不受到任何伤害（包括随机伤害、AOE）
- 不阻挡嘲讽效果

### 5.4 随从死亡结算

```
随从受到伤害 → 生命值 ≤ 0
  ↓
进入「待死亡」状态
  ↓
当前正在结算的效果全部执行完毕
  ↓
同时结算所有待死亡随从：
  ├── 触发亡语（Deathrattle）—— 按入场顺序
  ├── 触发「随从死亡时」效果
  ├── 随从从战场移除
  └── 进入坟场
```

**关键规则**：
- 伤害是**同时**结算的，但死亡在**当前效果结算完成后**统一处理
- 多个随从同时死亡时，亡语按这些随从的**入场顺序**依次触发
- 在亡语结算过程中可能再造成伤害、再产生死亡——无限嵌套，逐层结算

### 5.5 AOE（范围伤害）的结算顺序

```
打出 AOE 法术（如烈焰风暴）
  ↓
对敌方所有随从造成 N 点伤害
  ↓
伤害同时造成
  ↓
所有生命值 ≤ 0 的随从进入待死亡状态
  ↓
AOE 法术结算完毕，进入坟场
  ↓
待死亡随从的亡语触发（按入场顺序）
  ↓
死亡随从移除，进入坟场
```

**重要**：AOE 只影响**施放时**在场的随从。如果某个亡语召唤了新随从，这些新随从**不受**已施放的 AOE 影响（因为 AOE 已经结算完毕）。

### 5.6 过量伤害（Overkill）

- 当造成的伤害 > 目标当前生命值时，多余的伤害称为「过量伤害」
- 正常情况下，过量伤害不产生任何效果
- 某些卡牌（如「线缕」）会利用过量伤害产生额外效果
- 过量伤害可以触发「造成过量伤害时」效果

## 六、英雄与英雄技能系统

### 6.1 英雄

| 属性 | 说明 |
|------|------|
| **生命值** | 初始 30 点，降到 0 或以下则失败 |
| **护甲值** | 额外的生命值缓冲，先于生命值被消耗 |
| **攻击力** | 通常为 0；装备武器后获得武器攻击力；某些卡牌可临时赋予攻击力 |

**英雄的生命周期**：
- 初始英雄：由所选职业决定（如吉安娜、安度因等）
- 英雄牌：打出后替换英雄和英雄技能，但不重置生命值
- 护甲可以无限叠加
- 英雄的最大生命值没有硬性上限（护甲可超过 30）

### 6.2 英雄技能（Hero Power）

**基础规则**：
- 消耗：2 点法力水晶
- 使用限制：每回合 1 次
- 每回合只能使用自己英雄当前拥有的英雄技能
- 英雄技能不能跨越回合保留使用次数

**英雄技能的刷新**：
- 打出英雄牌后，英雄技能被替换，且**立即刷新可用状态**
- 如果你本回合已用过英雄技能，打出英雄牌后可以再次使用新的英雄技能
- 某些卡牌可以将英雄技能的法力消耗降为 0（如拉兹）或增加使用次数

**英雄技能的交互**：
- 英雄技能不是卡牌，不计入手牌数量
- 英雄技能不视为「打出法术」（不触发法力浮龙等效果，除非单独注明）
- 英雄技能可以被某些卡牌效果改变（如暗影形态、裁决者图哈特）

### 6.3 护甲（Armor）

- 护甲是英雄的额外生命值
- 受到伤害时，优先扣除护甲，护甲归零后才扣生命值
- 护甲可叠加无上限
- 部分卡牌（如盾牌猛击）与护甲值产生互动
- 护甲与生命值的区别：
  - 治疗只能恢复生命值，不能增加护甲
  - 「获得护甲」只增加护甲，不影响生命值
  - 某些卡牌只根据护甲值判断效果，某些根据生命值

## 七、完整对局流程

### 7.1 开局前 —— 构筑牌组

- 牌组共 **30 张牌**
- 同名普通牌最多 2 张，同名传说牌最多 1 张
- 牌组中只能包含中立卡牌和所选职业的卡牌
- 牌组在开局时随机洗牌（双方均不知道对方的牌序）

### 7.2 开局 —— 换牌阶段（Mulligan）

```
1. 抛硬币决定先后手（50/50 随机）
2. 先手玩家获得 3 张起始手牌
3. 后手玩家获得 4 张起始手牌 + 1 张幸运币
4. 双方同时进入换牌阶段：
   ├── 双方各自选择任意数量的手牌替换（点选卡片）
   ├── 被替换的牌洗回牌库
   ├── 从牌库顶补充等量的牌
   └── 换牌只发生一次，不可反复
5. 换牌阶段结束
6. 第 1 回合开始（先手玩家）
```

**换牌阶段的特殊规则**：
- 任务牌→必定出现在起始手牌中（如果是第 1 回合换上来的则不会触发此规则）
- 替换回去的牌可能再次被抽到（因为洗入了牌库）
- 换牌后的手牌数 = 换牌前的手牌数

### 7.3 游戏循环

```
当前玩家回合 {
  回合开始阶段 {
    获得/填满法力水晶
    抽牌
    触发回合开始效果
  }

  主要行动阶段 {
    while (玩家未点击结束回合) {
      玩家可以：
        - 打出卡牌
        - 攻击
        - 使用英雄技能
        - 使用表情
    }
  }

  回合结束阶段 {
    触发回合结束效果
    移除临时效果
    回合移交
  }
}

对方回合 (同上)

→ 循环直到一方英雄生命值 ≤ 0（含护甲）
```

### 7.4 胜利条件

| 条件 | 说明 |
|------|------|
| **主要胜利条件** | 将对方英雄的生命值（含护甲）降至 0 或以下 |
| **疲劳死亡** | 对方在牌库为空时抽牌，受到致命疲劳伤害 |
| **特殊胜利条件** | 某些卡牌提供额外胜利条件（如机械克苏恩、天启四骑士），但这是卡牌效果，非核心规则 |

### 7.5 手牌上限

- 手牌上限为 **10 张**
- 手牌已达 10 张时：
  - 抽牌阶段抽到的牌 → 直接烧毁（进入坟场，不触发任何效果）
  - 卡牌效果抽的牌 → 直接烧毁
  - 卡牌效果生成的牌（如发现）→ 直接烧毁
- 被烧毁的牌**不视为"被打出"或"被弃掉"**

### 7.6 牌库为空的规则

- 牌库为空时可以正常进行游戏
- 需要抽牌时触发疲劳（见 1.6 节）
- 涉及牌库操作的效果（如洗牌入牌库、从牌库召唤等）在牌库为空时：能执行多少执行多少，无法执行的部分跳过

### 7.7 回合时间

- 每个回合有时间限制（常规为 75 秒）
- 超时后回合自动结束
- 连续出牌会重置/延长部分倒计时
- 时间限制仅在 PvP 模式中生效（本文档可暂不关心动画时长细节）

## 八、关键词机制速查

> 下文为各关键词的**一句话定义**，完整精确规则见《炉石传说关键词与机制大全》。

| 关键词 | 一句话定义 |
|--------|-----------|
| **战吼（Battlecry）** | 随从从手牌打出时触发的效果 |
| **亡语（Deathrattle）** | 随从死亡时触发的效果 |
| **嘲讽（Taunt）** | 对方攻击时必须优先攻击此随从 |
| **冲锋（Charge）** | 随从打出当回合即可攻击（无视召唤失调） |
| **突袭（Rush）** | 随从打出当回合可攻击对方随从，但**不能攻击英雄** |
| **圣盾（Divine Shield）** | 吸收第一次受到的伤害，然后消失 |
| **潜行（Stealth）** | 不可被指定为目标（攻击/法术/英雄技能），攻击后潜行消失 |
| **风怒（Windfury）** | 每回合可攻击 2 次 |
| **吸血（Lifesteal）** | 造成的伤害同时为英雄恢复等量生命值 |
| **剧毒（Poisonous）** | 对此随从造成的任何伤害都会消灭目标随从 |
| **冻结（Freeze）** | 失去下一次攻击机会 |
| **沉默（Silence）** | 移除目标随从上的所有效果和增益 |
| **发现（Discover）** | 从随机 3 张符合条件的牌中选择 1 张 |
| **奥秘（Secret）** | 挂在英雄上的隐藏法术，满足条件时在对手回合触发 |
| **过载（Overload）** | 锁定下回合的 X 颗法力水晶 |
| **连击（Combo）** | 本回合已打出过其他牌后才触发额外效果 |
| **法术伤害（Spell Damage）** | 法术造成的伤害增加 N 点 |
| **激励（Inspire）** | 使用英雄技能后触发 |
| **进化（Adapt）** | 从 10 种进化选项中选择 1 种 buff |
| **回响（Echo）** | 打出后，在手牌中产生一张临时复制（当回合可无限打出） |
| **磁力（Magnetic）** | 可单独打出或与机械随从融合 |
| **双生法术（Twinspell）** | 打出后在手中产生一张不带双生关键词的复制 |
| **复生（Reborn）** | 第一次死亡后以 1 点生命值复活 |
| **祈求（Invoke）** | 触发迦拉克隆的效果升级，并触发职业祈求效果 |
| **流放（Outcast）** | 在手牌的最左或最右端时打出，触发额外效果 |
| **法术迸发（Spellburst）** | 施放法术后触发一次性效果 |
| **腐蚀（Corrupt）** | 在手牌中，当你打出一张费用更高的牌后，此卡变为腐蚀形态 |
| **暴怒（Frenzy）** | 第一次受伤存活后触发一次性效果 |
| **可交易（Tradeable）** | 可拖入牌库花费 1 费替换为牌库顶的牌 |
| **荣誉消灭（Honorable Kill）** | 造成恰好等于目标生命值的伤害时触发 |
| **巨型（Colossal）** | 打出时额外召唤附属部件，附属部件计入随从上限 |
| **注能（Infuse）** | 当友方随从死亡时注能，注能完成后卡牌变为增强形态 |
| **快枪（Quickdraw）** | 在抽到此牌的回合打出时触发额外效果 |

## 附录B：触发与事件钩子清单

此清单列举了所有**游戏事件**以及可能被触发的**效果类型**，用于程序实现时的事件系统设计。

### B.1 事件 → 触发的效果

```
事件：回合开始（StartTurn）
  → 随从的"回合开始时"效果
  → 奥秘检查（极少，但存在）
  → 武器/英雄牌的特殊效果

事件：回合结束（EndTurn）
  → 随从的"回合结束时"效果
  → 临时 buff 清除
  → 回合移交

事件：抽牌（DrawCard）
  → "抽牌时"效果（如炸弹牌）
  → 疲劳检查
  → 手牌上限检查

事件：打出随从（PlayMinion）
  → 战吼效果
  → "打出随从后"效果
  → 连击计数器更新
  → 法术迸发触发器（不适用，随从非法术）

事件：打出法术（PlaySpell）
  → 法术效果结算
  → "施放法术后"效果
  → 法术迸发触发器
  → 连击计数器更新

事件：攻击声明（DeclareAttack）
  → 冻结检查
  → "攻击前"效果
  → 奥秘检查（如冰冻陷阱）

事件：攻击结算（ResolveAttack）
  → 伤害结算
  → 圣盾检查
  → 剧毒检查
  → 吸血检查
  → "攻击后"效果
  → 武器耐久消耗

事件：随从受伤（MinionDamaged）
  → 暴怒检查
  → "受到伤害后"效果
  → 生命值检查 → 可能引发死亡

事件：随从死亡（MinionDeath）
  → 亡语触发
  → "随从死亡后"效果
  → 注能计数器
  → 坟场更新
  → 复活池更新

事件：英雄受伤（HeroDamaged）
  → 护甲优先扣除
  → 奥秘检查（如爆炸符文）
  → "英雄受到伤害后"效果

事件：使用英雄技能（HeroPower）
  → 英雄技能效果结算
  → 激励效果
  → "使用英雄技能后"效果

事件：回合开始——对方（OpponentStartTurn）
  → 己方奥秘检查（所有奥秘在此刻可能触发）

事件：回合结束——对方（OpponentEndTurn）
  → （通常无己方效果触发，极少数例外）
```

### B.2 事件优先级

在同时触发多个效果时，按以下优先级结算：

```
1. 当前正在结算的效果  →  必须完全执行完毕
2. 触发的新效果        →  按以下子优先级：
   a. 亡语（按随从入场顺序）
   b. "回合开始时/结束时"效果（按入场顺序）
   c. "造成伤害后"效果
   d. "打出后"效果
3. 状态检查           →  生命值 ≤ 0 的随从死亡
4. 衍生死亡效果       →  亡语触发 → 回到步骤 1
```

> 这个递归模型确保了无论嵌套多深，游戏状态始终一致。
