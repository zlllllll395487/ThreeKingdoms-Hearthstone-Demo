## 一、设计总纲与开发约束

### 1.1 设计哲学

```
炉石的核心骨架（保留 ~40%）
    +
三国世界观皮肤（命名 + 视觉）
    +
三国味道（仅 1 个新机制：火攻）
    ↓
三国炉石 v2.0  ——  一个人能做完的炉石变体
```

**与 v1.0 的根本区别**：v1.0 默认会"完整继承"炉石所有内容并扩展，目标过大；v2.0 把"砍内容"当作一等公民——明确写出**砍了什么、为什么砍**。

### 1.2 核心约束

本作以"独立开发可完成"为最高约束：

| 约束 | 数值 | 含义 |
|---|---|---|
| 团队规模 | 1-2 人 | 没有专门的策划/美术/前端/后端分工 |
| M0 周期 | 3 个月 | 第一个能跑通完整对局的 demo |
| M1 周期 | 6 个月 | 内测可用版本 |
| 单关键词实现成本 | < 2 天 | 否则砍掉 |
| 单卡牌效果文本 | ≤ 2 行 | 复杂效果留给传说 |

### 1.3 三阶段路线图

```
M0 (前 3 个月) —— 「能玩」
  ├─ 3 阵营（蜀/魏/吴），第 4 阵营群雄推迟到 M1
  ├─ 13 个核心关键词
  ├─ 40 张精选卡牌
  ├─ 仅 PvE（对战 AI）或本地热座 PvP
  └─ 仅 1 个新机制：火攻

M1 (第 4-6 个月) —— 「能内测」
  ├─ 加入第 4 阵营：群雄
  ├─ 加入 5 个进阶关键词：发现/暴怒/连击/复生/过载
  ├─ 卡池扩到 60-80 张
  ├─ 简单匹配系统（如有后端能力）
  └─ 基础卡牌平衡迭代

M2 (后续) —— 「能上线」
  ├─ 奥秘（锦囊）系统
  ├─ 任务（霸业）系统
  ├─ 英雄牌（名将牌）系统
  ├─ 在线天梯
  └─ 卡包/经济系统
```

**重要：M0 是项目成功的唯一硬指标**。M1/M2 都是「如果 M0 做完还有体力再说」。

### 1.4 命名

- 中文名：**三国炉石**
- 英文名：**Sanguo TCG**（不再用 Hearth of Three Kingdoms，避免商标风险）

## 二、阵营与英雄系统

### 2.1 四大阵营（M0 仅 3 个）

| 阵营 | 领袖 | 核心风格 | 一句话玩法 | M0 是否实装 |
|---|---|---|---|:---:|
| **蜀** | 刘备 | 铺场 + buff | 用低费小兵 + buff 滚雪球 | ✅ |
| **魏** | 曹操 | 兵器 + 护甲 | 攻防兼备，靠武器换血 | ✅ |
| **吴** | 孙权 | 过牌 + 计策 | 用过牌找关键计策爆发 | ✅ |
| **群雄** | 吕布 | 高攻快攻 | 高攻随从直接打脸 | ⏳ M1 |

> **不再用「炉石职业类比表」**：v1.0 的「蜀=圣骑+萨满」之类对照容易让玩家觉得是换皮。这种类比只用于内部团队沟通，不出现在对外文档里。

### 2.2 英雄技能（每个 2 费，每回合 1 次）

**设计原则**：**全部 1:1 映射到炉石现有英雄技能**，不发明新模板。这是 v2.0 最重要的妥协——独立开发没有平衡迭代的余裕，必须用炉石 10 年实战数据兜底。

| 阵营 | 技能名 | 效果 | 炉石原型 | 设计意图 |
|---|---|---|---|---|
| 蜀 · 刘备 | **仁德** | 召唤一个 1/1 义兵（步兵） | 萨满祭司「图腾召唤」简化版 | 持续铺场 |
| 魏 · 曹操 | **奸雄** | 你的英雄获得 +2 护甲 | 战士「护甲值」 | 防御反打 |
| 吴 · 孙权 | **制衡** | 抽 1 张牌 | 德鲁伊「皇帝召唤」简化版 / 牧师「次级治疗」改造 | 资源压制 |
| 群雄 · 吕布 | **武勇** | 对一个目标造成 1 点伤害 | 法师「火焰冲击」 | 主动控场 |

### 2.3 与 v1.0 的英雄技能差异

| v1.0 | 问题 | v2.0 改为 | 原因 |
|---|---|---|---|
| 蜀「仁德」+1/+1 | 比萨满图腾还强，单卡价值高 | 改为召唤 1/1 义兵 | 1:1 对应炉石职业模板 |
| 魏「奸雄」偷牌库顶 | 偷异色牌的规则未定义 | 改为 +2 护甲 | 直接复用炉石战士模板 |
| 吴「制衡」抽 1 / 抽 2 | 强度过高，会无限循环 | 改为单纯抽 1 | 直接复用炉石抽牌模板 |
| 群雄「无双」+2 攻 | 配兵器 = 暴力 5+ 攻 | 改为对目标 1 点伤害 | 复用法师火焰冲击 |

### 2.4 不做的内容

- ❌ **多英雄选择**：每个阵营固定 1 个英雄
- ❌ **名将牌（英雄牌）**：M2 再考虑
- ❌ **英雄皮肤**：完全砍

## 三、卡牌类型体系

### 3.1 仅 3 种类型（v1.0 是 6 种）

| 类型 | 对应炉石 | 说明 |
|---|---|---|
| **武将** | Minion | 上场战斗单位 |
| **计策** | Spell | 一次性法术 |
| **兵器** | Weapon | 装备给英雄的武器 |

**v1.0 中的「名将牌/锦囊/霸业」全部砍掉**：
- 名将牌（英雄牌）→ M2
- 锦囊（奥秘）→ M2
- 霸业（任务）→ M2

理由：奥秘需要"对手回合事件总线 + 隐藏信息可视化"，任务需要"多步触发条件 UI"，独立开发投入产出比极低。

### 3.2 兵种（M0 仅 3 种）

| 兵种 | 风格 | M0 实装 |
|---|---|:---:|
| **步兵** | 均衡攻血，铺场用 | ✅ |
| **骑兵** | 高攻低血 | ✅ |
| **谋士** | 低攻高血，与计策协同 | ✅ |
| 弓兵 | 远程，特殊攻击规则 | ⏳ M1 |
| 方士 | 治疗 / buff 专家 | ⏳ M1 |

> v1.0 设计了 5 个兵种但只占 3 张方士、5 张弓兵，严重失衡。M0 砍到 3 个，每个至少 8-10 张牌，确保兵种协同有牌可拼。

> **弓兵的「远程不参与直接战斗特殊规则」整体砍**——这是引入了一个全新攻击规则分支，调试成本太高。M1 加入弓兵时按普通武将处理。

### 3.3 武将（= 随从）

完全继承炉石随从规则。属性：费用 / 攻击 / 生命 / 兵种 / 关键词 / 效果文本。

### 3.4 计策（= 法术）

完全继承炉石法术规则。M0 不做奥秘、不做任务、不做特殊子类型。

### 3.5 兵器（= 武器）

完全继承炉石武器规则。

## 四、关键词系统

### 4.1 设计原则

**第一期只做炉石中"已被反复验证好用且实现简单"的关键词**，砍掉所有 v1.0 的扩展和新创关键词。

### 4.2 M0 关键词（13 个）

#### 战斗类（9 个）

| 炉石原名 | 三国名 | 一句话定义 |
|---|---|---|
| Taunt | **镇守** | 对方攻击时必须优先攻击此武将 |
| Charge | **冲锋** | 打出当回合即可攻击（含英雄） |
| Rush | **突袭** | 打出当回合可攻击武将，不能攻击英雄 |
| Stealth | **伏兵** | 不可被指定为目标，攻击后失效 |
| Divine Shield | **铁壁** | 吸收第一次受到的伤害 |
| Windfury | **连斩** | 每回合可攻击 2 次 |
| Poisonous | **淬毒** | 造成的任何伤害消灭目标武将 |
| Lifesteal | **饮血** | 造成伤害时恢复英雄等量生命 |
| Freeze | **困阵** | 失去下一次攻击机会 |

#### 触发类（3 个）

| 炉石原名 | 三国名 | 一句话定义 |
|---|---|---|
| Battlecry | **威名** | 从手牌打出武将时触发 |
| Deathrattle | **遗志** | 武将死亡时触发 |
| Spell Damage | **谋略** | 计策造成的伤害 +N |

#### 状态类（1 个）

| 炉石原名 | 三国名 | 一句话定义 |
|---|---|---|
| Silence | **离间** | 移除目标武将的所有效果与增益 |

### 4.3 v1.0 → v2.0 命名修正

| v1.0 名 | 问题 | v2.0 改为 |
|---|---|---|
| 突阵（冲锋）| 与突袭字形太近，玩家阅读混淆 | **冲锋**（直接复用炉石中文名）|
| 先登（突袭）| 字面 = 登城先锋，反而像冲锋 | **突袭**（直接复用炉石中文名）|
| 威慑（剧毒）| 三国语境"威慑"=震慑，与"造成伤害即消灭"语义无关 | **淬毒** |
| 夺气（吸血）| 与「士气」机制名称撞车 | **饮血** |
| 疑兵（冻结）| 疑兵之计 = 虚张声势，不是禁止行动 | **困阵** |
| 镇守、铁壁、连斩、威名、遗志、谋略、离间、伏兵 | 命名贴切 | **保留** |

### 4.4 砍掉的关键词（详细见附录 B）

| 类型 | 砍掉的内容 | 理由 |
|---|---|---|
| 触发类 | 连击、过载、激励、法术迸发、暴怒 | M1 选择性加入（推荐顺序：暴怒 > 连击 > 过载） |
| 状态类 | 奥秘（锦囊）、发现（求贤） | 奥秘 → M2；发现 → M1 |
| 扩展 | 复生、回响、可交易、双生、磁力、巨型、注能、流放、腐蚀、快枪、祈求、进化、荣誉消灭 | 全砍 |
| 新创 | 单挑、羁绊、士气 | 全砍（详见第五章和附录 B） |

**M0 关键词总数：13 个**（v1.0 设计了 24 个 + 4 个新创 = 28 个）。

## 五、三国专属机制

### 5.1 v1.0 新机制全数评估

v1.0 新创了 4 个三国专属关键词，v2.0 砍掉 3 个、保留并简化 1 个：

| v1.0 新机制 | 处置 | 理由 |
|---|:---:|---|
| 单挑（Duel）| ❌ 砍 | 与攻击规则的边界 case 太多（伏兵、镇守、连斩、困阵都需要单独定义交互），实现复杂度高、新手认知负担重 |
| 羁绊（Bond）| ❌ 砍 | 「五虎上将」需要 5 张传说同时在场，几乎不可能；分级羁绊会大幅增加构筑复杂度；与"独立开发"约束冲突 |
| 士气（Morale）| ❌ 砍 | 阈值光环（士气 ≥5/≤2）使前 5 回合多在 debuff 状态，新手体验差；与炉石护甲机制重叠，无独特价值；保留为**纯护甲**即可 |
| 火攻（Fire Attack）| ✅ 保留并简化 | 三国味道最浓的机制，不做泛化关键词，仅作为 2-3 张特定计策的特色效果 |

### 5.2 火攻（最终设计）

**定义**：火攻不是关键词，而是**特定卡牌的效果描述**。

**效果**：
> 「**火攻**：造成 N 点伤害；目标在下次回合开始时再受到 1 点伤害。」

**规则**：
- 灼烧伤害固定 1 点（M0 不做按费用缩放）
- 灼烧只触发 1 次，触发后消失
- 灼烧不可叠加（多次火攻只刷新触发回合，不累加伤害）
- 灼烧可被「离间」移除（如果挂在武将身上）
- 灼烧伤害**触发**「受到伤害时」效果（如未来加入的暴怒）
- 灼烧伤害**不触发**「受到攻击时」效果

**载体**：
- M0 仅 2-3 张计策卡使用此效果（如「火攻之策」「赤壁之火」）
- 不做火攻关键词、不做火攻系武将

### 5.3 砍掉的"次级"机制

- ❌ 攻城 / 守城模式（v1.0 5.2）：完全砍
- ❌ 兵种协同卡（如「友方骑兵 +1 攻」光环）：M1 选择性加入

## 六、战场规则

完全继承炉石规则，**无任何改动**：

- 每方 7 武将上限
- 武将按打出顺序从左到右排列
- 站位固定不自动填补
- 相邻效果存在
- 坟场机制（用于死亡查询）

> v1.0 提到「武将展示新增兵种图标 / 阵营颜色 / 关键词图标」——这属于 UI 设计而非玩法规则，本文档不涉及，留待 UI 文档处理。

## 七、对局流程

### 7.1 完全保留炉石的部分

| 系统 | 说明 |
|---|---|
| 回合交替制 | 双方轮流回合 |
| 法力水晶递增 | 1-10 颗 |
| 手牌上限 10 张 | 超过则烧毁 |
| 疲劳机制 | 抽不到牌时按 1/2/3... 累加伤害 |
| 牌组 30 张 | 同名普通牌 ×2，传说 ×1 |
| 换牌阶段 | 起始手牌可选择性洗回 |
| 先后手 | 先手 3 张牌，后手 4 张牌 + 军饷 |
| 胜利条件 | 英雄生命值 ≤ 0 |
| 英雄初始生命值 | 30 |

### 7.2 军饷（= 炉石「幸运币」）

- 重命名 + 规则修正
- 效果：「本回合获得 1 颗临时法力水晶」（与炉石幸运币完全一致）
- 属于计策类型
- **不可通过卡牌效果生成**（修正 v1.0 的 N10 运粮官设计漏洞）：军饷只在开局发给后手玩家，独立于卡池

### 7.3 换牌阶段的特殊处理（M0 简化）

M0 不做炉石的"任务牌必定起始手牌"逻辑（因为不做任务），换牌完全简单：起始手牌中任意张可换回牌库洗牌。

## 九、卡牌设计原则

### 9.1 硬约束

1. **单卡关键词上限**：普通/稀有 ≤ 1 个；史诗 ≤ 2 个；传说 ≤ 3 个
2. **战吼/亡语**：单卡只能选其一（特殊传说除外）
3. **效果文本长度**：普通/稀有 ≤ 1 行；史诗 ≤ 2 行；传说 ≤ 3 行
4. **数值上限**：8 费内单卡攻击力 ≤ 8（吕布是 8 费 6 攻 + 关键词组合，不破例）
5. **法术伤害**：单张计策不超过 5 点单体或 4 点 AOE（除特殊高费传说）

### 9.2 稀有度设计

| 稀有度 | 名称 | 特点 | 牌组数量 |
|:---:|---|---|:---:|
| 普通 | 士卒 | 白板或简单效果 | 2 |
| 稀有 | 将领 | 1 个中等强度效果 | 2 |
| 史诗 | 名将 | 独特机制 | 2 |
| 传说 | 无双 | 唯一性、高强度 | 1 |

### 9.3 阵营 vs 中立比例

- **牌组 30 张构成**：阵营专属 + 中立卡，无下限
- **中立卡数量**：M0 总卡池约 13-14 张中立 + 9 张/阵营 × 3 阵营 = 40 张
- **阵营专属卡只能用于该阵营牌组**（禁止跨阵营）

### 9.4 设计流程铁律

> 这是 v1.0 没有但 v2.0 必须强调的：

1. **任何新卡先与炉石现有卡比较** — 如果找不到对照，说明设计可能有问题
2. **传说卡谨慎引入新机制** — 新机制至少在 5 张普通/稀有/史诗上验证后才能传说化
3. **"羁绊"逻辑禁区** — 任何"X+Y+Z 同时在场"的设计都要砍
4. **测试用 AI 对战 100 局** — 任何新卡或调整都要先跑 100 局 AI 自动对战看胜率

## 附录B：被砍内容与原因

### B.1 砍掉的"系统级"内容

#### 锦囊（奥秘）系统
- **理由**：奥秘需要"对手回合事件总线 + 隐藏信息可视化 + 5 个挂载位 UI + 触发优先级排序"——独立开发投入产出比极低
- **替代方案**：M2 再做

#### 霸业（任务）系统
- **理由**：任务需要"多步触发条件追踪 + 完成度可视化 + 起始手牌强制规则"——破坏 M0 简洁性
- **替代方案**：M2 再做

#### 名将牌（英雄牌）系统
- **理由**：替换英雄 + 替换技能 + 护甲附加 + 战吼连锁——4 重逻辑叠加，bug 重灾区
- **替代方案**：M2 再做

### B.2 砍掉的三国专属新机制

#### 单挑（Duel）
- **致命问题**：与攻击规则的交互边界过多
  - 单挑 + 镇守：能否绕过？需要明确
  - 单挑 + 伏兵：伏兵能否被单挑指定？
  - 单挑 + 连斩：单挑算 1 次还是 2 次攻击？
  - 单挑 + 困阵：被困阵单位能否反击？
- **新手认知负担**：玩家学完攻击规则又要学单挑规则
- **可替代性**：单挑的"绕过镇守 + 模拟武将对决"完全可以用 1-2 张特定卡牌（如「张飞战吕布」战吼）实现，不需要做成关键词

#### 羁绊（Bond）
- **致命问题**：「五虎上将」需要 5 张传说同时在场——传说牌组限 1 张，5 张全凑齐 + 全活着 + 全没被沉默 = 概率接近 0
- **构筑陷阱**：玩家会被诱导拼凑羁绊牌组，但对手稍有 AOE 就全废，玩家挫败感强
- **分级版也不行**：分级羁绊（2/3/全员）需要 UI 实时显示进度，独立开发成本过高
- **可替代性**：用 1-2 张计策卡（如「桃园结义」单卡触发 buff）模拟即可

#### 士气（Morale）
- **致命问题**：士气 ≥5 / ≤2 的阈值光环 + 全场武将攻击力实时调整 = 频繁触发 + 频繁更新 + bug 风险高
- **新手体验差**：初始士气 = 0，前 5 回合常在 ≤2（debuff 状态）
- **可替代性**：直接用炉石护甲机制即可——曹操英雄技能就是 +2 护甲

### B.3 砍掉的扩展关键词（按炉石原名）

| 关键词 | 砍的原因 |
|---|---|
| 进化（Adapt）| 10 选 3 弹窗，UI 重 + 平衡黑洞 |
| 回响（Echo）| 临时复制需要"回合结束自动消失"的特殊清理逻辑 |
| 磁力（Magnetic）| 主题是机械，与三国不契合（v1.0 自己也指出了）|
| 双生法术（Twinspell）| 生成衍生卡的临时复制 + 永久复制混合，复杂 |
| 复生（Reborn）| M1 加入（实现简单但优先级低）|
| 祈求（Invoke）| 迦拉克隆专属，三国无对应 |
| 流放（Outcast）| 手牌位置判定，UI 复杂 |
| 腐蚀（Corrupt）| 手牌中的形态变化 + 触发追踪 |
| 可交易（Tradeable）| 拖拽到牌库的 UI 逻辑 + 1 费支付 |
| 荣誉消灭（Honorable Kill）| 精确数值匹配触发，玩家感知低 |
| 巨型（Colossal）| 附属部件 + 战场容量复杂交互 |
| 注能（Infuse）| 手牌中的事件追踪 |
| 快枪（Quickdraw）| 抽牌回合追踪 |
