本文档基于：
- 卡牌总数 **57 张**（蜀 25 / 吴 25 / 中立 7）
- Token 总数 **7 张**
- 套牌格式 **30 张** = 25 阵营 + 5 中立

### 1.1 📐 vanilla 公式与关键词税

#### 基线公式

```
单卡 mana value = (attack + health) ÷ cost
基线目标：2.0 ~ 2.2 mana value
低于 1.8：偏弱（适合需联动 / 极限 setup 的卡）
高于 2.4：偏强（必须通过条件 / 难度 / 反制平衡）
```

#### 关键词税表

| 关键词 | stat 调整 | 说明 |
|:--|:-:|:--|
| 镇守 (`taunt`) | −0.5 | 强迫敌方先攻击 |
| 突袭 (`rush`) | −0.5 | 当回合可打武将不能打英雄 |
| 冲锋 (`charge`) | −1.0 | 当回合可打任意目标 |
| 连斩 (`windfury`) | −1.0 | 每回合可攻击 2 次 |
| 神算 1 (`spellpower`) | −1.0 | 法术伤害 +1 |
| 战吼小效果 | −0.5 | 召唤 1 个低费 token / 单体小 buff |
| 战吼大效果 | −1.0 | 召唤多个 / AoE / 抽牌 |
| 亡语小效果 | −0.5 | 召唤 token |
| 亡语大效果 | −1.0 | AoE / 召唤多个 |
| 受击触发 | −0.5 | 每张每局 1 次（onceUsedKey 限制） |

#### 法术 / 兵器 价值换算

- **法术伤害**：1 mana = 2 dmg potential（条件性如 AoE 自伤 -0.5 mana value）
- **抽牌**：1 张 ≈ 1.5 mana value
- **治疗**：1 HP ≈ 0.4 mana value（HS vanilla 标准）
- **兵器 dmg potential**：attack × durability；1 mana = 2 dmg potential
- **法力返还**：1 mana refund ≈ 0.8 future mana value（不影响下回合上限）

#### v5.4 阵营差异化双轨制（用户拍板，废弃通用 1/2/3 算法）

> 用户：「吴国卡池里**只有 1 张** 1 费卡（W18 火油）。通用算法强制每局塞入 → 起手 100% 固化，丧失抽牌随机乐趣。必须废弃通用函数改双轨制。」

```
起手手牌组成（先手 3 张 / 后手 4 张 + 幸运币）：

蜀套牌：
- 强制换入 1 张 1 费（蜀 1 费有 S01/S02/S21/N06 共 4 张，随机塞 1）
- 强制换入 1 张 2 费（蜀 2 费有 S03/S04/S05/S25 共 4 张，随机塞 1）
- 剩余 1-2 张完全随机（先手 1 张随机 / 后手 2 张随机）

吴套牌：
- 跳过 1 费检查（吴 1 费 only W18 火油 → 强制塞入会 100% 固化）
- 强制换入 1 张 2 费（吴 2 费有 W08/W11/W14/W20/W22/W26 共 6 张）
- 强制换入 1 张 3 费（吴 3 费有 7-9 张可选）
- 剩余 1-2 张完全随机
- 设计理据：吴控制阵营第一回合空过是正常战略选择
```

📐 **替换算法 v5.4**：
```typescript
function ensureSmoothOpener(hand, deck, faction):
  const required = faction === 'shu'
    ? [1, 2]      // 蜀：保 1 费 + 2 费铺开
    : [2, 3]      // 吴：跳过 1 费，保 2 费 + 3 费打 setup

  for (const cost of required) {
    if (hand.some(c => c.cost === cost)) continue;
    const candidate = deck.find(c => c.cost === cost);
    if (candidate) {
      const removed = hand.maxByCost();  // 替换 hand 中最贵的
      hand.replace(removed, candidate);
      deck.placeAtBottom(removed);
    }
    // 容错：牌库中无该费用就跳过
  }
```

> ✅ **v5.4 风险审查通过**：双轨制保留吴起手随机性（1 费抽到 W18 火油 vs 抽不到 W18 各 ~50% 概率），符合「吴第一回合空过是战略选择」的阵营定位。蜀双费 1+2 保证铺开节奏。

### 3.2 v5.3 启发式打分公式（极简版）

> 用户：「用一个简单的权重字典（基于卡牌身材计算，并给具有"镇守"或核心锚点标签的目标增加基础权重分数）即可，不要引入复杂的搜索算法」

#### 📐 出牌阶段评分（CardPlayScore）

```typescript
score = (atk + hp)                       // base 身材
      + keywordBonus(card.keywords)      // 关键词加成
      + anchorSetupBonus(card)           // 锚点 setup 加成
      + comboSetupBonus(card)            // combo setup 加成
      + spellValue(card)                 // 法术效果价值

keywordBonus:
  taunt: +1
  rush:  +1.5
  charge: +2
  windfury: +1.5
  spellpower: +1（吴套牌内）/ 0（蜀套牌内）

anchorSetupBonus:
  if card 是锚点武将（W01/W02/W03）:
    +3（高优先级 setup）
  if card 联动卡 且 对应锚点在场:
    +5（联动激活，价值倍增）

comboSetupBonus:
  if card 设置 combo flag（W18/W20）:
    if 手牌中有对应后置卡（W19/W21）: +4
    else: +1
  if card 是后置卡 且 flag 已设: +3

spellValue:
  damage_spell: 总伤害 × 1.0
  draw_spell:
    if (deck.length === 0) return -1000  // v5.4 防自杀（疲劳期不抽）
    else: 抽牌数 × 1.5
  heal_spell:
    if (caster.hero.health === caster.hero.maxHealth) return -100  // 满血禁治疗
    else: 治疗量 × 0.4
  freeze/cannotAttack: 目标 atk × 0.8
```

> ✅ **v5.4 防自杀机制**：原公式中 draw_spell 无脑获得抽牌数 × 1.5 加分，残局牌库空时 AI 仍会打《苦肉计》自伤 2 抽 0 张 → 被疲劳抽死。新增 deck.length === 0 时强制返回 -1000，禁 AI 疲劳期抽牌。同时补丁满血禁治疗（避免 AI 无脑刷《金疮药》浪费 mana）。

#### 📐 攻击目标评分（AttackTargetScore）

```typescript
// v5.4 防过载（生存判定）顶部 guard
const enemyNextTurnAttack = sum(enemy.board.minions.map(m => m.atk))
if (enemyNextTurnAttack >= AI.hero.health) {
  // 生存模式：AI 残血时强制清场最高威胁
  return enemy.board.minions.maxBy(m => m.atk)
}

// 正常模式：常规威胁评估
targetScore =
  if 可斩杀敌方英雄: +1000  // 强制斩杀
  if 敌方 taunt: +500       // 强制先解
  if 敌方锚点武将（W01/W02/W03 on enemy side）: +50   // 优先杀锚点
  if 敌方高威胁（atk ≥ 4）: +30
  if 敌方关键威胁单位（S18/S19/S20 滚雪球 on enemy）: +40
  if 敌方法师辅助（spellpower 单位）: +20
  if 敌方普通武将（atk < 4）: +10
  if 敌方英雄（非斩杀但有 dmg potential）: +5
```

> ✅ **v5.4 防过载机制**：原公式锚点加 +50 / 高威胁 +30，AI 残血时仍会去解锚点忽略斩杀威胁导致下回合被反杀。**判例**：AI 3 HP，敌方有 5/5 白板 + 2/4 周瑜 → 原 AI 会优先解周瑜，下回合被 5/5 打死。新加生存判定（顶部 guard）：若敌方下回合总攻 ≥ AI HP，强制返回最高场攻单位（必须清场求生）。

📐 **decision rule**：在所有合法出牌 / 攻击中，选 score 最高的执行；若多个并列，选 mana cost 最高的（出牌阶段）/ 自身 hp 最高的（攻击阶段，保留低 hp 单位）

### 3.3 出牌阶段决策树（v5.3 扩展）

```
新增决策点（基于当前 Greedy 之上）：

1. 联动机会优先：
   if 锚点武将在场:
     for card in 手牌:
       if card.anchorRequirement matches in-board anchor:

2. combo setup 优先：

3. 锚点 setup 优先：

4. 回合内法力规划：

```

### 3.4 攻击阶段决策树

```
1. 斩杀检查：
   if 我方总 atk ≥ 敌方英雄 hp:
     all-in face

2. taunt 解除：
   if 敌方有 taunt:
     先用 atk 最高单位攻击 hp 最低 taunt

3. 锚点斩杀：
   if 敌方 W01 周瑜（破坏火烧赤壁联动）:

4. favorable trade:
   for each 我方 minion:
     for each 敌方 minion:
       if 我方 atk ≥ 敌方 hp AND 敌方 atk < 我方 hp:
         记录此 trade 为 favorable

5. 兜底打脸：
   if 无 favorable trade 且 无 taunt:
     全员攻击敌方英雄
```

#### checkWinner() 触发点（engine/index.ts:459-468）

```
function checkWinner():
  playerDead = player.hero.health ≤ 0
  aiDead = ai.hero.health ≤ 0
  if playerDead AND aiDead:
    // v5.3.1 拍板：PvE 平局规则 = 玩家强制胜利
    state.phase = 'ended'
    state.winner = 'player'  // 让玩家爽
    return
  if aiDead:
    state.phase = 'ended'
    state.winner = 'player'
    return
  if playerDead:
    state.phase = 'ended'
    state.winner = 'ai'
```
