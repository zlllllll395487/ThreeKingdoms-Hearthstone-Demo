## 3. 命名规范

成品资源遵循 `类别_主题[_变体].格式` 的小写蛇形命名：

```
cardvisual_guanyu.png        卡牌主视觉 · 关羽
portrait（在 portraits/）     武将立绘
btn_end_turn.png             按钮 · 结束回合
kw_taunt.png                 关键词印章 · 镇守
frame_legendary.png          边框 · 传说
tab_shu.png                  分页 · 蜀
tutorial_btn_long_on.png     教程 · 长按钮 · 开启态
```

- 全小写、以下划线分词，避免空格与中文文件名（成品层）。
- 同组资源用统一前缀，便于按前缀批量检索与管理。
- 开关 / 状态类资源以 `_on` / `_off` 等后缀区分变体。

## 4. 格式策略

| 资源类型 | 选用格式 | 理由 |
|:--|:--|:--|
| 大幅立绘 / 卡牌主视觉 | WebP（q=88） | 体积较 PNG 显著降低，视觉无明显损失 |
| 含透明通道的 UI 元素 | PNG | 保证透明边缘锐利 |
| 矢量图标 | SVG | 任意缩放无损 |
| 字库 | WOFF2 | Web 字体标准、压缩率高 |
| 动画 | MP4 | 通用视频格式 |

> 立绘与 Loading 背景的 WebP 化使该部分体积下降约 76%（详见 README §30）。
> WebP 转换前的原 PNG 备份仅保留于本地（`backup-original-png/`，已 `.gitignore`），便于回滚。
