### 2.3 用户感知 vs 数据反差（重要）

**用户多次反馈"吴方 OP，AI 用吴清场蜀方"**，但 iter6.1 数据显示：
- 蜀 vs 吴 60.0%（蜀优势）
- 吴 vs 蜀 51.6%（吴微弱优势）

**反差原因分析**：
1. 用户实际**用蜀方手动玩 / 用蜀方托管**对战吴方 AI
2. 蜀方需要"铺场滚雪球"，手操经验不足时不如 AI
3. 吴方 AoE（草船借箭 / 截江断流 / 火烧赤壁）一次清场体验冲击大，**心理感受** > 数据胜率
4. 中立 minion 全套（N09/N10/N11）让吴方"看起来兵多"，但实际是双方对等

**结论**：数值上蜀略 OP，但**体验上吴方"看起来"更强**。
**建议**：考虑增强用户的蜀方体验感（卡面/特效突出蜀方铺场快感），数值微调 3 张蜀方 OP 卡同步进行。

## § 8 · 必须修的三大问题（防屎山）

### 🔴 P0-1 · 启用 TypeScript strict 模式

**位置**：`game/tsconfig.app.json`

**修改**：
```json
{
  "compilerOptions": {
    "strict": true,
    "noImplicitAny": true,
    "strictNullChecks": true,
    ...
  }
}
```

**影响**：会出现 50-100 处类型错误，需逐个修复。预计 6-10 小时。

**收益**：长期可维护性 + 接手 AI 不易引入 bug + 89 处 `as` 大半可消灭。
