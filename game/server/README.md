# 在线对战后端 · 里程碑 0「连通性验证」

> 目标：用最小代价验证「WebSocket + 房间系统 + 引擎可在 Node 加载 + 跨设备连通」这条后端链路。
> 验证通过后再往上堆对战逻辑（里程碑 1+）。

## 这一步在验证什么

| 验证项 | 通过标准 |
|:--|:--|
| 引擎能在 Node 后端加载 | 启动日志打印「引擎模块加载成功，卡池 66 张」 |
| WebSocket 服务可用 | 测试页能连上、状态变「已连接」 |
| 房间系统 | A 建房得 4 位房间码，B 输码加入，A 收到「对手已加入」 |
| 跨设备实时连通 | A / B 互发消息，对方即时收到 |

引擎本身**零改动**，后端通过 `@/` 别名复用 `game/src/engine`（与 `scripts/sim/` 在 Node 跑引擎同理）。

## 启动后端

在 `game/` 目录下：

```bash
npm run server          # 单次运行
npm run server:watch    # 改代码自动重启（开发用）
```

启动后会打印三个地址：

```
[server] 测试页（本机）   http://localhost:8787
[server] 测试页（同 WiFi）http://172.20.x.x:8787  ← 手机连这个
[server] 内网穿透（跨网络）ngrok http 8787，把公网地址发给对手
```

## 三种验证方式（按从易到难）

### 方式 A · 同一台电脑两个标签页（最快，验证逻辑）

1. `npm run server`
2. 浏览器开两个标签，都打开 `http://localhost:8787`
3. 标签 1 点「建房」→ 得到房间码（如 `K7QR`）
4. 标签 2 在「输入房间码」填 `K7QR` → 点「加入」
5. 标签 1 应显示「★ 对手已加入房间」
6. 任一标签发消息，两边都即时收到 → **连通成功**

### 方式 B · 电脑 + 手机，同一个 WiFi（验证跨真实设备）

1. 电脑 `npm run server`，记下日志里的「同 WiFi」地址（如 `http://172.20.86.23:8787`）
2. 电脑浏览器打开 `http://localhost:8787` → 建房，得房间码
3. **手机连同一个 WiFi**，浏览器打开 `http://172.20.86.23:8787`（电脑那个地址）
   - 网址打开时，测试页会自动填好 ws 地址，无需手动配置
4. 手机输房间码 → 加入
5. 电脑收到「对手已加入」，互发消息验证

> 若手机连不上：多半是电脑防火墙挡了 8787 端口。Windows 会在首次启动时弹「是否允许」，选「允许」；或在「Windows Defender 防火墙 → 允许应用」里放行 node。

### 方式 C · 跨网络（手机用流量 / 朋友远程，验证真·公网穿透）

1. 电脑 `npm run server`
2. 另开一个终端，用内网穿透工具把 8787 暴露到公网，二选一：

   **ngrok**（需注册免费账号）：
   ```bash
   ngrok http 8787
   ```
   得到形如 `https://xxxx.ngrok-free.app` 的地址。

   **cloudflared**（无需账号，临时隧道）：
   ```bash
   cloudflared tunnel --url http://localhost:8787
   ```
   得到形如 `https://xxxx.trycloudflare.com` 的地址。

3. 把这个公网地址发给对手（或手机用流量打开）
4. 对手打开该地址 → 测试页会自动用 `wss://` 连接（https 隧道下浏览器要求 wss）
5. 建房 / 加入 / 互发消息验证

> 跨网络是真·在线对战的部署形态：你电脑跑后端，隧道暴露公网，任何人都能连入。
> 代价：你电脑必须开着。等里程碑都跑通、想做 7×24 公开服务时，再把后端迁到云托管（Railway / Fly.io 等）。
